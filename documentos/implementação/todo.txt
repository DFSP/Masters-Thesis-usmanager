Na replicação, ver se existem containers parados no host, se sim, então reiniciar esses em vez de começar um container novo
1. Adicionado uma funcionalidade a todos os microserviços para que se deregistem no registration-server ao terminar a sua execução

No host onde está a executar o container
2. Parar o container
	docker stop $CONTAINER
3. Create a new image
	docker commit $CONTAINER $CONTAINER
4. Guardar e carregar a imagem no novo host
	docker save $CONTAINER | ssh $USER@$HOST docker load 

5. Guardar os volumes do container (use ".tar.gz" if you want compression)
	docker-volumes.sh $CONTAINER save $CONTAINER-volumes.tar

6. Copiar os volumes para o novo host
	scp $CONTAINER-volumes.tar $USER@$HOST:

No host para onde vai migrar o container:

7. Criar  novo container com as mesmas opções que tinha o container anterior
	docker create --name $CONTAINER [<PREVIOUS CONTAINER OPTIONS>] $CONTAINER

8. Carregar os volumes
	docker-volumes.sh $CONTAINER load $CONTAINER-volumes.tar

9. Começar o container # Start container
docker start $CONTAINER
